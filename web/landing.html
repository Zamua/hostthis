HOSTTHIS(1) User Commands HOSTTHIS(1)

Name

hostthis — pipe HTML or Markdown to ssh, get a public URL.

Synopsis

cat file | ssh hostthis.dev [slug] [--name "label"]
ssh hostthis.dev verb [args]

Description

A dev-first paste service for content that needs rendering to be shareable — HTML and Markdown today; mermaid and friends later. Pipe a file to ssh, get a URL on stdout. Every paste lives at its own subdomain (<slug>.hostthis.dev) so the browser sandbox isolates each one. Every paste expires 24 hours after its last update.

Your ssh key is your account — no signup, no password, no UI. Without a key you can still upload anonymously; you just can’t list, update, or delete your pastes from another machine. Public-key snooping gives an attacker nothing — ssh requires proving you hold the private key.

Examples

$ cat index.html | ssh hostthis.dev
https://abc12345.hostthis.dev
expires in 24h

$ cat README.md | ssh hostthis.dev --name "Project README"
https://x7y8z9q0.hostthis.dev
"Project README" — expires in 24h

$ cat v2.html | ssh hostthis.dev abc12345    # update (24h clock resets)
$ ssh hostthis.dev list                       # your active pastes
$ ssh hostthis.dev delete abc12345             # permanent

Commands

cat file | ssh hostthis.dev [--name label]
upload a new paste (random slug)
cat file | ssh hostthis.dev slug
update an existing paste; resets the 24h clock
ssh hostthis.dev list
your active pastes, soonest-to-expire first
ssh hostthis.dev show slug
read content back over ssh (owner only)
ssh hostthis.dev rename slug "label"
set or change a paste’s label; empty string to clear
ssh hostthis.dev versions slug
history within the 24h window
ssh hostthis.dev pin slug ver
set which version the URL serves; does not reset the clock
ssh hostthis.dev unpublish slug
URL returns 404 to the public; owner can still show and link
ssh hostthis.dev publish slug
undo unpublish
ssh hostthis.dev link slug --expires 24h
issue a signed share URL for an unpublished paste
ssh hostthis.dev unshare slug
revoke all signed share links for a paste
ssh hostthis.dev delete slug
permanent; no confirmation prompt
ssh hostthis.dev whoami
your identity and quota
ssh hostthis.dev token create
issue an HTTP API token (for environments without ssh)

Accepted formats

HTML and Markdown. Per-paste cap is 5 MB. Anything else (zips, images, binaries) gets rejected with a pointer to a real file host. The acceptance gate exists so the sandbox has a small attack surface.

Security

Every paste lives at its own subdomain — cookies, JS, and CSP from abc.hostthis.dev can’t reach xyz.hostthis.dev or the apex. The default CSP is strict: scripts are same-origin only, connect-src is self so exfiltration channels are closed. Markdown is server-side sanitized before rendering so uploaded Markdown can never execute JS.

Limits

Per-paste hard cap: 5 MB.
Per quota class: 200 MB rolling 30-day window; 50 MB/day rate.
Retention: 24 hours from last update, then deleted.

The quota class is the equivalence class formed by union of your ssh key fingerprint and your IP subnet. Two requests share a quota if they share either the key or the subnet — to escape the class an attacker must rotate both simultaneously.

For AI agents

A thorough plain-text how-to lives at /llms.txt. Any LLM that can curl can use hostthis.
